# CLI 公開仕様（Windows）

## 対象範囲

本書は `cbz-viewer.exe` の公開 CLI インターフェースを定義する。

公開入力としてサポートするのは以下のみとする。

* パス指定なし
* パス指定 1 件（ファイルまたはディレクトリ）

それ以外のオプションは内部実装向けであり、本仕様の対象外とする。

---

## 起動例

### ライブラリモード（パス指定なし）

```powershell
cbz-viewer.exe
```

### ライブラリモード（ディレクトリ指定）

```powershell
cbz-viewer.exe <directory-path>
```

### ビューアモード（ファイル指定）

```powershell
cbz-viewer.exe <file-path>
```

---

## 起動ルール

### 1. パス指定なし

ライブラリモードで起動する。

---

### 2. ディレクトリを 1 件指定

* ライブラリモードで起動する
* 指定したディレクトリを初期表示フォルダとして開く
* 画像を直下に持つ FolderBook でも、CLI 上は通常のディレクトリ入力として扱う

---

### 3. ファイルを 1 件指定

以下の条件を満たす必要がある。

* ファイルが存在すること
* 対応アーカイブ形式、または対応画像形式であること

条件を満たす場合、ビューアモードで起動する。

* アーカイブファイルは、その本を直接開く
* 画像ファイルは、親フォルダを開き、指定画像の位置から開始する

---

### 4. 複数パス指定

エラーとして終了する。

---

## パスの取り扱い

* Windows 専用仕様とする
* パス区切り文字として `\` と `/` の両方を受け付ける
* パス正規化はライブラリ画面のパス入力と同じ処理を使用する

対象実装:

```text
src/app/platform.rs::normalize_dir_path
```

---

## エラー処理

入力が不正な場合は起動に失敗し、プロセスを終了する。

エラー情報は以下へ出力する。

* stderr
* ログ出力

GUI エラーダイアログの表示は本仕様では規定しない。
