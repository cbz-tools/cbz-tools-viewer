# 本のグループ設定

## グループとは

* Library左ペインで本を絞り込むための分類機能です。
* 1冊に直接設定できるグループ名は1つです。
* `groups` で親子関係を作ると、親グループから子グループをまとめて扱えます。
* お気に入りとは別機能です。
* 現在の設定対象は Archive です。FolderBook / ImageFile / Folder は対象外です。

### 対象

| 種別 | 扱い |
| --- | --- |
| Archive | 設定対象 |
| FolderBook | 対象外 |
| ImageFile | 対象外 |
| Folder | 対象外 |

`画集C` のような FolderBook は、現状のグループ設定対象外です。

## UIから設定する

1. Libraryで対象の Archive を選びます。
2. 右クリックして「グループを設定...」を選びます。
3. 既存グループから選ぶか、手入力します。
4. 確定します。

複数選択は `Ctrl+クリック`、`Shift+クリック`、`Ctrl+A` が使えます。

複数選択中に右クリックした場合は、選択中の Archive がまとめて対象になります。
Archive 以外には設定できません。

「未分類に戻す」で個別指定を解除できます。

## TOMLで設定する

保存先は `%LOCALAPPDATA%/cbz-viewer/kind_groups.toml` です。

ファイルがない場合は、初回起動時にテンプレートが作成されます。

```toml
[overrides]
"C:\\path\\to\\作品A 第01巻.cbz" = "シリーズA"
"C:\\path\\to\\作品A 第02巻.zip" = "シリーズA"
"C:\\path\\to\\作品B 総集編.rar" = "未整理"
"C:\\path\\to\\短編集D.cbz" = "短編集"

[[kind_rules]]
# ファイル名の先頭にある (シリーズA) のような Kind に対して正規表現で一致します。
pattern = "^シリーズA$"
group = "シリーズA"

[[kind_rules]]
pattern = "^画集$"
group = "画集"

[groups."まとめ"]
children = ["シリーズA", "画集", "短編集"]
```

`[overrides]` はパスの完全一致です。
アプリが正規化したパスで照合します。

`[[kind_rules]]` はファイル名の先頭 Kind に対する正規表現です。
上から順に見て、最初に一致したものが使われます。

`[groups."..."]` は親グループです。
`children` に書いたグループをまとめて扱います。

`kind_rules` に当たらなくても Kind がある本は、その Kind 文字列がグループ名になります。
Kind がなく、`overrides` も当たらない本が未分類です。

## 優先順位

```text
1. UIから設定した個別指定（[overrides]）
2. TOMLの [[kind_rules]]（上から順に最初に一致したもの）
3. ファイル名の Kind 文字列そのもの
4. どれにも当たらない場合は未分類
```

`[groups]` は優先順位ではなく、絞り込みと集計のための親子関係です。

## グループを解除する

* UIの「未分類に戻す」を押すと、その本の個別指定を削除します。
* 個別指定を削除したあと、`kind_rules` または Kind の自動判定に戻ります。
* どちらにも当たらなければ未分類になります。

## 設定が反映されない場合

* `kind_rules` の正規表現が Kind に一致していない
* ファイル名に Kind がない
* UIの個別指定が残っている
* TOMLの構文エラー、未定義 child、循環参照がある
* 外部編集は数秒後に再読込されるので、少し待つかアプリを開き直す
